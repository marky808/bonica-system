# BONICA 納品履歴CSVエクスポート機能 実装指示書

## 概要

納品履歴データをCSV形式でエクスポートする機能を追加する。
前シーズンの納品実績集計などに使用。

---

## 機能要件

### 1. UIの追加場所

納品一覧画面（既存）に「CSVダウンロード」ボタンを追加

### 2. フィルター条件

以下の条件でフィルタリングできるようにする：

- **期間指定**（必須）
  - 開始日（date picker）
  - 終了日（date picker）
- **納品先**（任意）
  - 既存の納品先マスタから選択
  - 「すべて」も選択可能
- **商品カテゴリー**（任意）
  - 既存カテゴリーから選択
  - 「すべて」も選択可能

### 3. CSV出力項目

以下の項目を出力する（この順番で）：

| 項目名 | 内容 |
|--------|------|
| 納品日 | YYYY-MM-DD形式 |
| 納品番号 | システム内部ID |
| 納品先 | 納品先名称 |
| 商品名 | 商品名 |
| カテゴリー | 商品カテゴリー |
| 数量 | 数値 |
| 単位 | 単位名 |
| 単価 | 税抜単価 |
| 税率 | 8%または10% |
| 金額（税抜） | 数量×単価 |
| 金額（税込） | 税込金額 |
| 仕入れID | 紐付いている仕入れID（ない場合は空欄） |
| 備考 | 備考欄 |

### 4. ファイル名

`納品履歴_YYYYMMDD_YYYYMMDD.csv`

（例：納品履歴_20240401_20240930.csv）

### 5. 文字コード

UTF-8 with BOM（Excelで文字化けしないように）

---

## 技術仕様

### API エンドポイント

```
GET /api/deliveries/export
```

**クエリパラメータ：**
- `startDate`: 開始日（YYYY-MM-DD）
- `endDate`: 終了日（YYYY-MM-DD）
- `customerId`: 納品先ID（optional）
- `categoryId`: カテゴリーID（optional）

### 実装方針

1. 既存の納品一覧取得ロジックを流用
2. Prismaでフィルター条件付きクエリ
3. サーバーサイドでCSV生成
4. Content-Disposition: attachment でダウンロード

### フロントエンド

- 既存の納品一覧画面にフィルターUIを追加
- 日付選択はdate-fnsまたは既存のdate pickerを使用
- 「CSVダウンロード」ボタン押下でAPIを呼び出し
- ローディング表示あり

---

## 注意点

- 大量データ対策：1回の出力上限は10,000件とする（超える場合はエラーメッセージ表示）
- 空データ対策：該当データがない場合は「該当するデータがありません」とアラート
- 既存機能への影響：既存の納品一覧画面の動作は変更しない
